API Documentation

PixelProbe REST API

The PixelProbe API provides programmatic access to all media scanning and management functionality. All API endpoints return JSON responses.

Base URL

https://your-server.com/api

Authentication

Currently, the API does not require authentication. In production environments, you should implement proper authentication.

Core Endpoints

GET /api/version

Get the current application version and GitHub URL.

Response Example:
{ "version": "1.47", "github_url": "https://github.com/ttlequals0/PixelProbe" }
GET /health

Health check endpoint for monitoring.

Response Example:
{ "status": "healthy" }
GET /api/stats

Get summary statistics of all scanned files.

Response Example:
{ "total_files": 10542, "corrupted_files": 245, "healthy_files": 9897, "warning_files": 400, "pending_files": 0, "scanning_files": 0, "marked_as_good": 150 }
cURL Example:
curl -X GET "https://your-server.com/api/stats"
GET /api/system-info

Get detailed system information including monitored paths and scan statistics.

Response Example:
{ "database_stats": { "total_files": 10542, "corrupted_files": 245, "healthy_files": 9897, "files_by_path": { "/media/photos": 5234, "/media/videos": 5308 } }, "monitored_paths": [ { "path": "/media/photos", "exists": true, "file_count": 5234, "is_accessible": true } ], "scan_stats": { "total_scans": 156, "average_scan_time": 12.5, "last_scan_date": "2024-01-15T10:30:00Z" }, "current_scan": { "is_scanning": false, "scan_id": null } }

Scanning Endpoints

POST /api/scan-all

Start a full scan of all configured media paths.

Response Example:
{ "message": "Scan started", "scan_id": "scan_123456" }
POST /api/scan-file

Scan a specific file by its path with optional deep scan.

Request Body:
Parameter Type Description
file_path string The full path of the file to scan
deep_scan boolean Enable enhanced corruption detection (optional, default: false)
Request Example:
{ "file_path": "/media/videos/movie.mp4", "deep_scan": true }
cURL Example:
curl -X POST "https://your-server.com/api/scan-file" \ -H "Content-Type: application/json" \ -d '{"file_path": "/media/videos/movie.mp4", "deep_scan": true}'
POST /api/scan-parallel

Scan multiple files in parallel with optional deep scanning.

Request Body:
Parameter Type Description
file_paths array[string] Array of file paths to scan
deep_scan boolean Enable enhanced corruption detection (optional, default: false)
GET /api/scan-status

Get the current scan status and progress information.

Response Example:
{ "is_scanning": true, "scan_id": "scan_123456", "total_files": 1000, "completed_files": 450, "pending_files": 550, "scanning_files": 5, "scan_phase": "scanning", "current_file": "video_file.mp4", "eta": 300, "scan_state_message": "Scanning: video_file.mp4 (45s)" }
POST /api/cancel-scan

Cancel the currently running scan operation.

Response Example:
{ "message": "Scan cancelled", "cancelled_files": 550 }

File Management Endpoints

GET /api/scan-results

Get paginated scan results with filtering and sorting options.

Query Parameters:
Parameter Type Description
page integer Page number (default: 1)
per_page integer Items per page (default: 50, max: 500)
sort_field string Field to sort by (default: scan_date)
sort_order string Sort order: asc or desc (default: desc)
filter string Filter: all, corrupted, warnings, healthy
search string Search in file paths
GET /api/scan-results/{result_id}

Get details for a specific scan result.

Response Example:
{ "id": 123, "file_path": "/media/videos/movie.mp4", "file_type": "video", "scan_status": "corrupted", "scan_date": "2024-01-15T10:30:00Z", "scan_output": "Error: Invalid codec parameters", "file_size": 1073741824, "file_hash": "abc123def456", "marked_as_good": false }
GET /api/scan-output/{result_id}

Get the full scan output for a specific result.

Response Example:
{ "scan_output": "[mov,mp4,m4a,3gp,3g2,mj2 @ 0x7f8b8c000000] moov atom not found\nInput #0, mov,mp4,m4a,3gp,3g2,mj2, from '/media/videos/movie.mp4':\n Duration: N/A, bitrate: N/A\nError: Invalid data found when processing input" }
POST /api/mark-as-good

Mark files as verified good (false positives).

Request Body:
Parameter Type Description
file_ids array[integer] Array of file IDs to mark as good
GET /api/view/{result_id}

Stream a media file for viewing (images and videos).

GET /api/download/{result_id}

Download a media file.

GET POST /api/export-csv

Export scan results to CSV format. GET exports all results, POST can export specific files.

Request Body (POST only):
Parameter Type Description
file_ids array[integer] Specific file IDs to export
cURL Example (Export All):
curl -X GET "https://your-server.com/api/export-csv" -o scan_results.csv
cURL Example (Export Specific Files):
curl -X POST "https://your-server.com/api/export-csv" \ -H "Content-Type: application/json" \ -d '{"file_ids": [123, 456, 789]}' \ -o selected_results.csv

Maintenance Endpoints

POST /api/reset-for-rescan

Reset files for rescanning based on criteria.

Request Body:
Parameter Type Description
reset_type string Type: "unscanned", "errors", or "all"
POST /api/reset-stuck-scans

Reset files stuck in 'scanning' status.

POST /api/cleanup-orphaned

Remove database entries for files that no longer exist on disk.

GET POST /api/file-changes

Check for files that have changed since last scan (based on modification time).

Response Example:
{ "message": "File changes check started", "task_id": "file_changes_123456" }
GET /api/cleanup-status

Get the status of the cleanup operation.

Response Example:
{ "is_cleaning": true, "cleanup_phase": "checking", "total_files": 1000, "checked_files": 500, "orphaned_files": 25, "cleanup_message": "Checking file existence: 500/1000" }
POST /api/cancel-cleanup

Cancel the running cleanup operation.

POST /api/reset-cleanup-state

Reset stuck cleanup state (emergency recovery).

GET /api/file-changes-status

Get the status of the file changes check operation.

POST /api/cancel-file-changes

Cancel the running file changes check operation.

POST /api/recover-stuck-scan

Emergency recovery for stuck scans.

Configuration Endpoints

GET /api/configurations

Get all configuration settings.

POST /api/configurations

Update configuration settings.

Request Body:
Parameter Type Description
key string Configuration key
value string Configuration value
GET /api/ignored-patterns

Get list of ignored error patterns.

Response Example:
[ { "id": 1, "pattern": ".*moov atom not found.*", "description": "Ignore moov atom errors for MP4 files", "created_at": "2024-01-15T10:30:00Z" }, { "id": 2, "pattern": ".*Non-monotonous DTS.*", "description": "Ignore DTS timestamp warnings", "created_at": "2024-01-16T11:00:00Z" } ]
POST /api/ignored-patterns

Add a new ignored error pattern.

Request Body:
Parameter Type Description
pattern string Regular expression pattern to match errors
description string Description of the pattern (optional)
Request Example:
{ "pattern": ".*Invalid NAL unit size.*", "description": "Ignore NAL unit size errors in H.264 streams" }
DELETE /api/ignored-patterns/{pattern_id}

Delete an ignored error pattern.

cURL Example:
curl -X DELETE "https://your-server.com/api/ignored-patterns/1"

Code Examples

Python Example

import requests import json # Base URL for the API BASE_URL = "https://your-server.com/api" # Start a full scan def start_scan(): response = requests.post(f"{BASE_URL}/scan-all") return response.json() # Get scan status def get_scan_status(): response = requests.get(f"{BASE_URL}/scan-status") return response.json() # Get scan results with filtering def get_scan_results(page=1, filter="corrupted"): params = { "page": page, "per_page": 50, "filter": filter } response = requests.get(f"{BASE_URL}/scan-results", params=params) return response.json() # Mark files as good def mark_files_as_good(file_ids): data = {"file_ids": file_ids} response = requests.post(f"{BASE_URL}/mark-as-good", json=data) return response.json() # Example usage if __name__ == "__main__": # Start scan scan_result = start_scan() print(f"Scan started: {scan_result}") # Check status status = get_scan_status() print(f"Scan status: {status}") # Get corrupted files results = get_scan_results(filter="corrupted") print(f"Found {results['total']} corrupted files")

JavaScript/Node.js Example

const axios = require('axios'); const BASE_URL = 'https://your-server.com/api'; // Start a full scan async function startScan() { try { const response = await axios.post(`${BASE_URL}/scan-all`); return response.data; } catch (error) { console.error('Error starting scan:', error); } } // Get scan results with pagination async function getScanResults(page = 1, filter = 'all') { try { const response = await axios.get(`${BASE_URL}/scan-results`, { params: { page, per_page: 50, filter } }); return response.data; } catch (error) { console.error('Error getting results:', error); } } // Scan specific files with deep scan async function scanFiles(filePaths, deepScan = true) { try { const response = await axios.post(`${BASE_URL}/scan-parallel`, { file_paths: filePaths, deep_scan: deepScan }); return response.data; } catch (error) { console.error('Error scanning files:', error); } } // Example usage (async () => { // Start full scan const scanResult = await startScan(); console.log('Scan started:', scanResult); // Get results const results = await getScanResults(1, 'corrupted'); console.log(`Found ${results.total} corrupted files`); // Scan specific files const files = ['/media/video1.mp4', '/media/video2.mp4']; await scanFiles(files, true); })();

Bash/cURL Example

#!/bin/bash BASE_URL="https://your-server.com/api" # Function to start a scan start_scan() { curl -X POST "$BASE_URL/scan-all" } # Function to check scan status check_status() { curl -X GET "$BASE_URL/scan-status" | jq '.' } # Function to get corrupted files get_corrupted_files() { curl -X GET "$BASE_URL/scan-results?filter=corrupted&per_page=100" | jq '.' } # Function to export results to CSV export_to_csv() { curl -X GET "$BASE_URL/export-csv" -o "scan_results_$(date +%Y%m%d).csv" echo "Results exported to scan_results_$(date +%Y%m%d).csv" } # Function to cleanup orphaned files cleanup_orphaned() { echo "Starting cleanup..." curl -X POST "$BASE_URL/cleanup-orphaned" # Monitor cleanup status while true; do STATUS=$(curl -s "$BASE_URL/cleanup-status" | jq -r '.is_cleaning') if [ "$STATUS" = "false" ]; then echo "Cleanup completed" break fi sleep 2 done } # Example workflow echo "Starting PixelProbe scan..." start_scan # Wait for scan to complete while true; do IS_SCANNING=$(curl -s "$BASE_URL/scan-status" | jq -r '.is_scanning') if [ "$IS_SCANNING" = "false" ]; then echo "Scan completed" break fi sleep 5 done # Get results echo "Fetching corrupted files..." get_corrupted_files # Export to CSV export_to_csv

PowerShell Example

$BaseUrl = "https://your-server.com/api" # Start a full scan function Start-PixelProbeScan { $response = Invoke-RestMethod -Uri "$BaseUrl/scan-all" -Method Post return $response } # Get scan status function Get-ScanStatus { $response = Invoke-RestMethod -Uri "$BaseUrl/scan-status" -Method Get return $response } # Get scan results function Get-ScanResults { param( [int]$Page = 1, [string]$Filter = "all" ) $params = @{ page = $Page per_page = 50 filter = $Filter } $response = Invoke-RestMethod -Uri "$BaseUrl/scan-results" -Method Get -Body $params return $response } # Mark files as good function Set-FilesAsGood { param( [int[]]$FileIds ) $body = @{ file_ids = $FileIds } | ConvertTo-Json $response = Invoke-RestMethod -Uri "$BaseUrl/mark-as-good" -Method Post -Body $body -ContentType "application/json" return $response } # Example usage Write-Host "Starting PixelProbe scan..." $scan = Start-PixelProbeScan Write-Host "Scan ID: $($scan.scan_id)" # Monitor scan progress do { Start-Sleep -Seconds 5 $status = Get-ScanStatus Write-Host "Progress: $($status.completed_files)/$($status.total_files)" } while ($status.is_scanning) # Get corrupted files $corrupted = Get-ScanResults -Filter "corrupted" Write-Host "Found $($corrupted.total) corrupted files"
